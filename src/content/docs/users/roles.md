Drizby uses CASL for role-based access control. Roles determine what actions a user can perform.

## Built-in Roles

- **Admin** — Full access to all features, settings, and user management.
- **Editor** — Can create and edit dashboards, notebooks, schemas, and cubes.
- **Viewer** — Can view dashboards and run notebook queries.

Permissions are enforced on both the API and UI layers.
